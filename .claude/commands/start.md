Follow the startup routine at the beginning of each session:

1. Read `.claude/CLAUDE.md` for project context
2. Read `.claude/WORK_LOG.md` for work history
3. Read `.claude/SESSION_HANDOFF.md` for last session details
4. Check `git status` for any uncommitted changes
5. Review recent `git log` to understand latest work
6. Provide a concise session summary including:
   - Current project state
   - Recent accomplishments (from commits)
   - Any uncommitted work
   - Relevant context from previous session
7. Ask the user what they'd like to work on this session
